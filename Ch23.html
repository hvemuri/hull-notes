Estimating Volatility

Let σn\sigma_nσn​ be volatility of a market variable on day nnn as estimated at the end of day n−1n-1n−1.
The square of the volatility σn2\sigma_n^2σn2​ is the variance rate on day nnn.

Suppose the value of the market variable at the end of day iii is SiS_iSi​.
The variable uiu_iui​ is the continuously compounded return during day iii.

ui=ln⁡SiSi−1u_i = \ln \frac{S_i}{S_{i-1}}ui​=lnSi−1​Si​​

An unbiased estimate of the variance rate per day using the most recent mmm observations on the uiu_iui​ is given by

σn2=1m−1∑i=1m(un−i−uˉ)2\sigma_n^2 = \frac{1}{m-1} \sum_{i=1}^m (u_{n-i} - \bar{u})^2σn2​=m−11​i=1∑m​(un−i​−uˉ)2

where uˉ\bar{u}uˉ is the mean of the uiu_iui​.

To monitor daily volatility, the formula is changed in three ways:

  1. uiu_iui​ is defined as the percentage change in the market variable

ui=Si−Si−1Si−1u_i = \frac{S_i - S_{i-1}}{S_{i-1}}ui​=Si−1​Si​−Si−1​​

  1. uˉ\bar{u}uˉ is assumed to be zero
  2. m−1m-1m−1 is replaced by mmm (moves from unbiased estimate to maximum likelihood estimate)

The simplified formula for the variance rate is

σn2=1m∑i=1mun−i2\sigma_n^2 = \frac{1}{m} \sum_{i=1}^m u_{n-i}^2σn2​=m1​i=1∑m​un−i2​

Weighting Schemes

The equation above gives equal weight to un−12,un−22,…,un−m2u_{n-1}^2, u_{n-2}^2, \dots, u_{n-m}^2un−12​,un−22​,…,un−m2​. Since the objective is to estimate the current level of volatility, it makes sense to give more weight to more recent data.

σn2=∑i=1mαiun−i2\sigma_n^2 = \sum_{i=1}^m \alpha_i u_{n-i}^2σn2​=i=1∑m​αi​un−i2​

where αi\alpha_iαi​ is the weight given to the observation iii days ago.

The α\alphaα values should be positive, increasing (less weight to older days), and sum to 1.

An extension of this is to assume there is a long-run average variance rate and that this should be given more weight.

σn2=γVL+∑i=1mαiun−i2\sigma_n^2 = \gamma V_L + \sum_{i=1}^m \alpha_i u_{n-i}^2σn2​=γVL​+i=1∑m​αi​un−i2​

where VLV_LVL​ is the long-run variance rate and γ\gammaγ is the weight assigned to VLV_LVL​.

The weights must sum to 1 so γ+∑i=1mαi=1\gamma + \sum_{i=1}^m \alpha_i = 1γ+i=1∑m​αi​=1

ARCH(m) model (autoregressive conditional heteroscedasticity)

σn2=ω+∑i=1mαiun−i2\sigma_n^2 = \omega + \sum_{i=1}^m \alpha_i u_{n-i}^2σn2​=ω+i=1∑m​αi​un−i2​ where ω=γVL\omega = \gamma V_Lω=γVL​.

The Exponentially Weighted Moving Average Model

EWMA model (exponentially weighted moving average)

  • particular case of the model (given above) σn2=∑i=1mαiun−i2\sigma_n^2 = \sum_{i=1}^m \alpha_i u_{n-i}^2σn2​=i=1∑m​αi​un−i2​
  • weights decrease exponentially going back through time αi+1=λαi     where     0<λ<1\alpha_{i+1} = \lambda \alpha_i \,\,\,\,\, \text{where} \,\,\,\,\, 0 < \lambda < 1αi+1​=λαi​where0<λ<1

This leads to a formula for updating volatility estimates

σn2=λσn−12+(1−λ)un−12\sigma_n^2 = \lambda \sigma_{n-1}^2 + (1-\lambda)u_{n-1}^2σn2​=λσn−12​+(1−λ)un−12​

  • λ\lambdaλ governs how responsive the estimate of daily volatility is to the most recent daily percentage change
    – low λ\lambdaλ means a great deal of weight is given to the un−12u_{n-1}^2un−12​ term
    – high λ\lambdaλ means the estimate responds slowly to new information provided by daily change

The GARCH(1, 1) Model

GARCH(1, 1) model (generalized autoregressive conditional heteroscedasticity)

σn2\sigma_n^2σn2​ is calculated from a long-run average variance rate VLV_LVL​ as well as from σn−1\sigma_{n-1}σn−1​ and un−1u_{n-1}un−1​

σn2=γVL+αun−12+βσn−12\sigma_n^2 = \gamma V_L + \alpha u_{n-1}^2 + \beta \sigma_{n-1}^2σn2​=γVL​+αun−12​+βσn−12​

where γ\gammaγ, α\alphaα, and β\betaβ are weights with

γ+α+β=1\gamma + \alpha + \beta = 1γ+α+β=1

The EWMA model is a particular case of GARCH(1, 1) where γ=0\gamma = 0γ=0, α=1−λ\alpha = 1-\lambdaα=1−λ, and β=λ\beta = \lambdaβ=λ.

The (1, 1) indicates that σn2\sigma_n^2σn2​ is based on the most recent observations of u2u^2u2 and the most recent estimate of the variance rate (σ2\sigma^2σ2).

GARCH(1, 1) can also be written

σn2=ω+αun−12+βσn−12\sigma_n^2 = \omega + \alpha u_{n-1}^2 + \beta \sigma_{n-1}^2σn2​=ω+αun−12​+βσn−12​

where ω=γVL\omega = \gamma V_Lω=γVL​.

Once ω\omegaω, α\alphaα, and β\betaβ have been estimated, calculate γ\gammaγ as 1−α−β1 - \alpha - \beta1−α−β, and calculate VLV_LVL​ as ω/γ\omega / \gammaω/γ.

Note that α+β<1\alpha + \beta < 1α+β<1 so that γ\gammaγ (weight applied to long-term variance) is not negative.

The Weights

Note that the model is recursive.
Substituting for σ\sigmaσ shows that the weight applied to un−i2u_{n-i}^2un−i2​ is αβi−1\alpha \beta^{i-1}αβi−1.

The weights decline exponentially at rate β\betaβ, which is the decay rate

  • similar to λ\lambdaλ in the EWMA model
  • relative importance of the observations on the uiu_iui​ in determining current variance rate
    – Let β=0.9\beta = 0.9β=0.9
    – un−22u_{n-2}^2un−22​ is 90% as important as un−12u_{n-1}^2un−12​
    – un−32u_{n-3}^2un−32​ is 81% as important as un−12u_{n-1}^2un−12​

GARCH(1, 1) model is similar to EWMA model

  • both have weights that decline exponentially for past u2u^2u2
  • GARCH(1, 1) also assigns some weight to the long-run average volatility

Mean Reversion

GARCH(1, 1) model recognizes that the variance approaches the long-run average level VLV_LVL​

The weight assigned to VLV_LVL​ is γ\gammaγ.

GARCH(1, 1) is equivalent to a model where the variance VVV follows the stochastic process

dV=a(VL−V)dt+ξV dzdV = a(V_L - V) dt + \xi V \, dzdV=a(VL​−V)dt+ξVdz

where time is measured in days, a=1−α−βa = 1 - \alpha - \betaa=1−α−β, and ξ=α2\xi = \alpha \sqrt{2}ξ=α2​.

This is a mean-reverting model

  • variance has a drift that pulls it back to VLV_LVL​ at rate aaa
  • when V>VLV > V_LV>VL​, the variance has negative drift
  • when V<VLV < V_LV<VL​, the variance has positive drift
  • superimposed on the drift is a volatility ξ\xiξ

Choosing Between the Models

Variance rates tend to be mean reverting in practice. GARCH(1, 1) model incorporates mean reversion. EWMA model does not. So GARCH(1, 1) is theoretically more appealing.

When ω=0\omega = 0ω=0, GARCH(1, 1) reduces to EWMA.
When ω<0\omega < 0ω<0, GARCH(1, 1) is not stable, so EWMA makes more sense.

Maximum Likelihood Methods

  • how parameters in the models are estimated from historical data
  • involves choosing values for the parameters that maximize the chance of the data occurring

Example

Suppose 10 stocks are sampled at random on a certain day.
The price of one declined on that day, and the prices of the others remained or increased.

Let probability of price decline be ppp.
Probability that a particular stock declines in price and the others do not is P=p(1−p)9P = p(1-p)^9P=p(1−p)9.
The best estimate of ppp is the one that maximizes PPP.

dPdp=(1−10p)(1−p)8=0\frac{dP}{dp} = (1-10p)(1-p)^8 = 0dpdP​=(1−10p)(1−p)8=0

⇒p=0.1\Rightarrow p = 0.1⇒p=0.1

So the maximum likelihood estimate of ppp is 0.10.10.1

Estimating a Constant Variance

∏i=1m[12πvexp⁡(−ui22v)]\prod_{i=1}^m \left[ \frac{1}{\sqrt{2\pi v}} \exp \left( \frac{-u_i^2}{2v} \right) \right]i=1∏m​[2πv​1​exp(2v−ui2​​)]

Maximizing an expression is equivalent to maximizing the logarithm of the expression.

Differentiating this expression with respect to vvv and setting it equal to zero gives

How Good is the Model?

The assumption underlying GARCH is that volatility changes with the passage of time

  • some periods have high volatility (when ui2u_i^2ui2​ is high, ui+12,ui+22,…u_{i+1}^2, u_{i+2}^2, \dotsui+12​,ui+22​,… tend to be high)
  • some periods have low volatility (when ui2u_i^2ui2​ is low, ui+12,ui+22,…u_{i+1}^2, u_{i+2}^2, \dotsui+12​,ui+22​,… tend to be low)

This can be tested be examining the autocorrelation structure of the ui2u_i^2ui2​.

Assume the ui2u_i^2ui2​ exhibit autocorrelation. If GARCH model is working, it should remove the autocorrelation. Test this by considering the autocorrelation structure for the variables ui2/σi2u_i^2 / \sigma_i^2ui2​/σi2​.
If these show little autocorrelation, the model for σi\sigma_iσi​ has succeeded in explaining autocorrelations in the ui2u_i^2ui2​.

A more scientific test is the Ljung-Box statistic.

If a certain series has mmm observations then the Ljung-Box statistic is

m∑k=1Kwkηk2m \sum_{k=1}^K w_k \eta_k^2mk=1∑K​wk​ηk2​

where ηk\eta_kηk​ is the autocorrelation for a lag of kkk, KKK is the number of lags considered, and

wk=m+2m−kw_k = \frac{m+2}{m-k}wk​=m−km+2​

Using GARCH(1, 1) to Forecast Future Volatility

The variance rate for day nnn using GARCH(1, 1) is

σn2=(1−α−β)VL+αun−12+βσn−12\sigma_n^2 = (1-\alpha-\beta)V_L + \alpha u_{n-1}^2 + \beta \sigma_{n-1}^2σn2​=(1−α−β)VL​+αun−12​+βσn−12​

so that

σn2−VL=α(un−12−VL)+β(σn−12−VL)\sigma_n^2 - V_L = \alpha(u_{n-1}^2 - V_L) + \beta(\sigma_{n-1}^2 - V_L)σn2​−VL​=α(un−12​−VL​)+β(σn−12​−VL​)

On day n+tn+tn+t in the future

σn+t2−VL=α(un+t−12−VL)+β(σn+t−12−VL)\sigma_{n+t}^2 - V_L = \alpha(u_{n+t-1}^2 - V_L) + \beta(\sigma_{n+t-1}^2 - V_L)σn+t2​−VL​=α(un+t−12​−VL​)+β(σn+t−12​−VL​)

The expected value of un+t−12u_{n+t-1}^2un+t−12​ is σn+t−12\sigma_{n+t-1}^2σn+t−12​ so

E[σn+t2−VL]=(α+β)E[σn+t−12−VL]E[\sigma_{n+t}^2 - V_L] = (\alpha + \beta) E[\sigma_{n+t-1}^2 - V_L]E[σn+t2​−VL​]=(α+β)E[σn+t−12​−VL​]

Using the equation repeatedly yields

E[σn+t2−VL]=(α+β)t(σn2−VL)E[\sigma_{n+t}^2 - V_L] = (\alpha + \beta)^t (\sigma_n^2 - V_L)E[σn+t2​−VL​]=(α+β)t(σn2​−VL​)

E[σn+t2]=VL+(α+β)t(σn2−VL)E[\sigma_{n+t}^2] = V_L + (\alpha + \beta)^t (\sigma_n^2 - V_L)E[σn+t2​]=VL​+(α+β)t(σn2​−VL​)

This equation forecasts volatility on day n+tn+tn+t using information available at end of day n−1n-1n−1.

In the EWMA model α+β=1\alpha + \beta = 1α+β=1 so the expected future variance rate equals the current variance rate, which is consistent with previous results.

When α+β<1\alpha + \beta < 1α+β<1 the final term in the equation becomes smaller as ttt increases. The variance rate exhibits mean reversion with reversion level VLV_LVL​ and reversion rate 1−α−β1-\alpha-\beta1−α−β. So the forecast of future variance rate tends towards VLV_LVL​ as ttt increases.

When α+β>1\alpha + \beta > 1α+β>1 the weight given to the long-term average variance is negative, so the process is mean fleeing rather than mean reverting, and the process is not stable.

Volatility Term Structures

Suppose it is day nnn.

Define

V(t)=E(σn+t2)V(t) = E(\sigma_{n+t}^2)V(t)=E(σn+t2​)

and

a=ln⁡1α+βa = \ln \frac{1}{\alpha + \beta}a=lnα+β1​

so that the previous equations becomes

V(t)=VL+e−at[V(0)−VL]V(t) = V_L + e^{-at} [V(0) - V_L]V(t)=VL​+e−at[V(0)−VL​]

In this equation V(t)V(t)V(t) is an estimate of the instantaneous variance rate in ttt days.

The average variance rate per day between today and time TTT is given by

1T∫0TV(t) dt=VL+1−e−aTaT[V(0)−VL]\frac{1}{T} \int_0^T V(t) \, dt = V_L + \frac{1 - e^{-aT}}{aT}[V(0) - V_L]T1​∫0T​V(t)dt=VL​+aT1−e−aT​[V(0)−VL​]

As TTT increases, this approaches VLV_LVL​.

Let σ(T)\sigma(T)σ(T) be the volatility per annum that should be used to price a TTT-day option under GARCH(1, 1). Assuming 252 days per year, σ(T)2\sigma(T)^2σ(T)2 is 252 times the average variance rate per day.

σ(T)2=252(VL+1−e−aTaT[V(0)−VL])\sigma(T)^2 = 252 \left( V_L + \frac{1-e^{-aT}}{aT} [V(0) - V_L] \right)σ(T)2=252(VL​+aT1−e−aT​[V(0)−VL​])

Recall that the market prices of different options on the same asset are used to calculate a volatility term structure. This is the relationship between the implied volatilities of the options and their maturities. The equation above can be used to estimate a volatility term structure based on the GARCH(1, 1) model.

When current volatility is above the long-term volatility, the GARCH(1, 1) model estimates a downward-sloping volatility term structure.

When current volatility is below the long-term volatility, the GARCH(1, 1) model estimates an upward-sloping volatility term structure.

Impact of Volatility Changes

When instantaneous volatility increases, the impact on volatility of an option is largest for short-life options and smallest for long-life options. Financial institutions use this when determining the exposure of their books to volatility changes. When calculating vega, they relate the size of the volatility increase to the maturity of the option.

Consistency Condition for Covariances

A covariance matrix can be constructed after calculating all variances and covariances.

Not all covariance matrices are internally consistent.

The consistency condition for an N×NN \times NN×N covariance matrix Ω\OmegaΩ is

ω⊺Ωω≥0\omega^\intercal \Omega \omega \ge 0ω⊺Ωω≥0

for all N×1N \times 1N×1 vectors ω\omegaω.

A matrix that satisfies this property is positive-semidefinite.

To ensure a positive-semidefinite matrix is produced, variances and covariances should be calculated consistently. This means if variances are calculated by giving equal weight to the last mmm data items, then the same should be done for covariances. If using the EWMA model, then the λ\lambdaλ parameter should be the same for calculating variances and covariances.

Example of a covariance matrix that is not internally consistent

[100.9010.90.90.91] \begin{bmatrix} 1 & 0 & 0.9\\ 0 & 1 & 0.9\\ 0.9 & 0.9 & 1\\ \end{bmatrix} ⎣⎡​100.9​010.9​0.90.91​⎦⎤​

  • the variance of each variable is 1.01.01.0 and the covariances are coefficients of correlation
  • the first variable is highly correlated with the third variable
  • the second variable is highly correlated with the third variable
  • however there is no correlation between the first and second variables

Set ω=(1,1,−1)\omega = (1, 1, -1)ω=(1,1,−1) to show that the matrix is not positive-semidefinite.
